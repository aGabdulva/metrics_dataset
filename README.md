# metrics_dataset

В данном проекте мною реализована витрина данных(датасет в Redash) для дальнейшей реализации выскороуревного дашборда для CPO.
Для реализации витрины был написан SQL-скрипт(PostrgeSQL), который извлекает данные по необходимым метрикам в нужных разрезах, с возможностью подсчета YoY/MoM/WoW.
Стоит отметить, что данные в изначальных табличках "сырые", то есть напрямую метрики из них не извлечь, поэтому в скрипте также производится расчет каждой необходимой метрики.
Скрипт расположен в файле `script.sql`.

**Необходимые разрезы:**

*  Метрика
*  Категория
*  Город

**Необходимые метрики**

* Выручка
*  Кол-во байеров
*  Кол-во проданных айтемов
*  Кол-во заказов
*  Кол-во возвратов
* Время доставки

**Описание данных**

Данные лежат в табличках

* public_aaa.aaa_bi_items
* public_aaa.aaa_bi_dict_item_categories
* public_aaa.aaa_bi_item_price
* public_aaa.aaa_bi_orders
* public_aaa.aaa_bi_order_delivery_time
* public_aaa.aaa_bi_order_items
* public_aaa.aaa_bi_customers

Описание таблиц и полей можно найти в файле `description.xslx` или по [ссылке](https://docs.google.com/spreadsheets/d/1zjldZyAdhIAQG8xyjJ_35IRicgr8AavupSRJFlH9cMc/edit?usp=sharing).

Примечание: скриншоты витрины не включены в данный проект, так как вставлять скриншоты витрины из Redash запрещено ввиду NDA.
